#HTML ##HyperText Markup Language
##HTML overview What is this extra content that we see in the text editor but not in the browser? It's HTML! HTML (HyperText Markup Language) was created for web browsers to read and interpret the meaning of the content. To display content properly in the browser, use *tags*. Left (`<`) & (`>`) right angled brackets surround the tag name. Tags usually come in pairs â€” an opening tag and a *closing tag with a forward slash*: ```xml <tagname></tagname> ``` Tags are used to describe the meaning of the content, not what they look like. That's what CSS is for. **Hint:** `shift + comma key` for the left angled bracket & `shift + period key` for the right angled brackets
##HTML tags Here are some tags already included in your project's `index.html`. **Title**: Found in the `<head>` section of the HTML document. Used for defining the page title. Displays in the browser's tab, search engine results and bookmarks. ```xml <title>Page Title</title> ``` **Headings**: There are 6 heading tags (`<h1>,<h2>,...,<h6>`) and they follow a hierarchical structure. `<h1>` represents the most important heading on the page. ```xml <h1>Most Important Heading</h1> <h2>Second Most Important Title</h2> ``` **Paragraphs**: Created with the `<p></p>` tags. ```xml <p>I'm a paragraph.</p> ``` **Block Quotation**: Created with the `<blockquote></blockquote>` tags. Used to indicates that the enclosed text is a quotation. A source of the quotation may be given using the `<cite>` tag. ```xml <blockquote> <p>You know you're in love when you can't fall asleep because reality is finally better than your dreams.</p> <cite>― Dr. Seuss</cite> </blockquote> ```
##HTML tags Here are a few others you may choose to add later today. **Link**: requires the addition of an `href` *attribute* to specify where to link to. ``` <a href="http://website.com">link</a> ``` **Lists**: There are two kinds of lists: ordered (`ol`) for numbered lists & undordered (`ul`) for lists with bullets. Content must be included only within the list item (`li`). ```xml <ul> <li>Unordered list item</li> <li>Unordered list item</li> </ul> <ol> <li>Ordered list item</li> <li>Ordered list item</li> </ol> ``` **Images**: like links, also requires the an additional attribute, `src`, for linking to the image file. ``` <img src="path/to/image.jpg"> ``` More resources here: [Mozilla Developer Network](https://developer.mozilla.org/en-US/docs/Web/), [HMTL5Doctor](http://html5doctor.com/element-index/)

#CSS ##Cascading Style Sheets
##CSS Syntax & Terminology Though there are some default styles attached to various HTML tags, it should be used to define the *content*. CSS defines the *presentation* and is the language used to make your site look any way you want! **Selectors** are used to determine which element(s) to apply the styles to. **Declarations** are written using `property:value` pairs and contained within a pair curly braces `{}`. ```css selector { property: value; } ``` **Properties** determine the type of style to be applied to the element (e.g. color) and follow by a colon (`:`). **Values** are specific to the property. Must end with a semi-colon (`;`). ```css h1 { font-size: 40px; } ```
##Comments Comments are great for leaving notes for yourself (or others), organizing your code blocks or for "commenting" out your code (hiding it for later). Note they syntax below: ``` p { color: #B109AA; /* LLC purple */ } /* HEADER STYLES */ header { width: 80%; } /* Saving this for later .slider { width: 100px; background: red; } */ ```
##Where Does the CSS Go? You can add CSS to your web page in many ways. Today we're going to put the CSS in it's own document, in a linked/external style sheet. Referenced in the `<head>` element with a `<link>` tag and two **attributes**, `rel` and `href`. The value of the `href` points to the location of the style sheet. ``` <head> <title>Page Title</title> <meta charset="utf-8"> <link rel="stylesheet" href="style.css"> </head> ``` There's already a `styles.css` file in your project folder. Let's link it in our HTML page. **Pro tip!** Sublime Text shortcut: type "link" + tab key. Ta da!
##Class & ID attributes When **type selectors** are too general (targeting the HTML element), HTML gives us two ways to attach extra identifying information to target specific elements with two **attributes**, `class` (can be used multiple times) and `id` (can only be used once per page). ``` /* targets ALL parapraphs */ p { color: blue; } /* only applies to the element that contains this class, defined by leading period */ .danger { color: red; } /* only applies to the element that contains this id, defined by number symbol */ #danger { border: 1px solid red; } ``` ```xml <p>Regular text.</p> <p class="danger">Red error message text or some other dangerous text.</p> <div id="danger">Will display a red border.</div> ```

##CSS Customizations The CSS for this template is written in [Sass](http://sass-lang.com/). For your forked copy of these slides, feel free to use either Sass or the compiled CSS files. Just note that if you pull in any styles updates from the repo, the CSS files will be recompiled. If you want to change the intro slide background image (or not use an image at all), change the style using the `.intro` class. The text in the slide can automatically scale based on window size. If you want to turn this feature "on" or off", in **framework > sass > _variables.scss** under font sizes, use the `rem` values for scalable text and `px` values for non-scalable text. (You can still zoom in with the pixel sized text.) Avoid changing the base styles files (`_mixins.scss`, `_variables.scss` and `theme-llc.scss`). Put all of your customizations and workshop specific styles in workshop.scss. If you'd like to add an improvement or enhancement, please submit a pull request or email [christina@ladieslearningcode.com](mailto:christina@ladieslearningcode.com).
