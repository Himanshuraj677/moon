#HTML ##HyperText Markup Language
##HTML overview What is this extra content that we see in the text editor but not in the browser? It's HTML! HTML (HyperText Markup Language) was created for web browsers to read and interpret the meaning of the content. To display content properly in the browser, use *tags*. Left (`<`) & (`>`) right angled brackets surround the tag name. Tags usually come in pairs — an opening tag and a *closing tag with a forward slash*: ```xml <tagname></tagname> ``` Tags are used to describe the meaning of the content, not what they look like. That's what CSS is for. **Hint:** `shift + comma key` for the left angled bracket & `shift + period key` for the right angled brackets
##HTML tags Here are some tags already included in your project's `index.html`. **Title**: Found in the `<head>` section of the HTML document. Used for defining the page title. Displays in the browser's tab, search engine results and bookmarks. ```xml <title>Page Title</title> ``` **Headings**: There are 6 heading tags (`<h1>,<h2>,...,<h6>`) and they follow a hierarchical structure. `<h1>` represents the most important heading on the page. ```xml <h1>Most Important Heading</h1> <h2>Second Most Important Title</h2> ``` **Paragraphs**: Created with the `<p></p>` tags. ```xml <p>I'm a paragraph.</p> ``` **Block Quotation**: Created with the `<blockquote></blockquote>` tags. Used to indicates that the enclosed text is a quotation. A source of the quotation may be given using the `<cite>` tag. ```xml <blockquote> <p>You know you're in love when you can't fall asleep because reality is finally better than your dreams.</p> <cite>― Dr. Seuss</cite> </blockquote> ```

#CSS ##Cascading Style Sheets
##CSS Syntax & Terminology Though there are some default styles attached to various HTML tags, it should be used to define the *content*. CSS defines the *presentation* and is the language used to make your site look any way you want! **Selectors** are used to determine which element(s) to apply the styles to. **Declarations** are written using `property:value` pairs and contained within a pair curly braces `{}`. ```css selector { property: value; } ``` **Properties** determine the type of style to be applied to the element (e.g. color) and follow by a colon (`:`). **Values** are specific to the property. Must end with a semi-colon (`;`). ```css h1 { font-size: 40px; } ```
##Comments Comments are great for leaving notes for yourself (or others), organizing your code blocks or for "commenting" out your code (hiding it for later). Note they syntax below: ``` p { color: #B109AA; /* LLC purple */ } /* HEADER STYLES */ header { width: 80%; } /* Saving this for later .slider { width: 100px; background: red; } */ ```
##Where Does the CSS Go? You can add CSS to your web page in many ways. Today we're going to put the CSS in it's own document, in a linked/external style sheet. Referenced in the `<head>` element with a `<link>` tag and two **attributes**, `rel` and `href`. The value of the `href` points to the location of the style sheet. ``` <head> <title>Page Title</title> <meta charset="utf-8"> <link rel="stylesheet" href="style.css"> </head> ``` There's a `styles.css` file in your project folder. Let's link it to the HTML page and see what happens! **Pro tip!** Sublime Text shortcut: type "link" + tab key. Ta da!

Padding and Margin

The padding and margin can both be declared using similar syntax of 1 to 4 values. Values can be declared using different measurement units but pixels or percentages are most commonly used. Setting the value to "0" will remove any default space on an HTML element. Any positive value will add space.

  • 4 values means top, right, bottom, left
  • 3 values means top, right & left, bottom
  • 2 values means top & bottom, right & left
  • 1 value means the same amount of padding on all sides

Change some values to see the difference between margin and padding. Trying using 1 - 4 values. Let's also inspect the element using Chrome's Developer tool.

#CSS3 Animations!
##Vendor Prefixes CSS vendor prefixes are used to add browser support for new and/or experimental features. Each browser uses its own prefix to render its own version of that property until the property becomes standardized across all browsers. Android, iOs, Chrome & Safari: `-webkit-` Internet Explorer: `-ms-` Firefox: `-moz-` Opera: `-o-` The prefix gets applied to the property name like so: `-webkit-property-name` `-moz-property-name` `-ms-property-name` `-o-property-name`
##Vendor Prefixes Vendor prefixes are temporary. As newer versions of browsers are released and these properties become part of the official, specifications, the prefixes are dropped. To future proof your CSS, list the standard property last. Browsers will ignore the properties it doesn't understand and move onto the next. ```css -webkit-border-radius: 50%; border-radius: 50%; ``` Currently, only the standard `border-radius` is needed. Including it last will ensure the property gets implemented in the event that the specifications have changed after you've written the CSS rule. **Extra Resources:** * [http://caniuse.com](http://caniuse.com) - reference * [http://pleeease.io/play/](http://pleeease.io/play/) - tool * [http://css-tricks.com/how-to-deal-with-vendor-prefixes/](http://css-tricks.com/how-to-deal-with-vendor-prefixes/) - extra reading
##CSS3 @keyframes To make a heart pulse effect, we need to find a way to tell the browser, make this element bigger, then smaller, then bigger, then smaller... To set multiple points for different transitions, use the `@keyframes` rule which includes an animation name, breakpoints and the properties to be animated. To create different breakpoints for the animations, use percentages (0% refers to the beginning of the animation, 100% is the end of an animation). @keyframes animation-name { 0% { width: 100px; } 50% { width: 200px; } 100% { width: 100px; } }
**Don't forget to check for prefix requirements!** @-webkit-keyframes animation-name { ... } @keyframes animation-name { ... } **Note** that animations can include multiple properties but the animation can only be applied to the same property, **not** from one property to another. For example, animating from `top: 0;` to `bottom: 0;` will not work. It must be from `top: 0;` to `top: 200px` or any other desired value but using the same property.
##CSS3 Transform The `transform` property can be used to add many effects like `skew`, `rotate` and [many more](http://learn.shayhowe.com/advanced-html-css/transitions-animations/). Today we are going to use `scale` to create a pulsing heart by making it bigger and smaller. Let's use that to create our animation. ```css @-webkit-keyframes pulse { 0% { transform: scale(1); } 100% { transform: scale(1.5); } } @keyframes pulse { 0% { transform: scale(1); } 100% { transform: scale(1.5); } } ```
##Using the @keyframe animation The `@keyframes` rule just *creates* the animation but to use it, it much be assigned to an element and a couple more properties need to be declared to determine how the animation should play out. The `animation-name` property is required as well as the `animation-duration` property so the browser knows which animation should be executed and for how long an animation should take to complete. By default, animations run **once** from beginning to end. To repeat an animation, use the `animation-iteration-count` property. The values can be a number or the `infinite` keyword to repeat the animation indefinitely. .to-be-animated { animation-name: pulse; animation-duration: 2s; animation-iteration-count: infinite; } This can also be written in shorthand. .to-be-animated { animation: pulse 2s infinite; }
##Class Exercise Let's put this all together and apply this pulse effect to our heart. First, add a class name to specify which heart element to apply this effect to. Let's call it `heart-pulse`. The CSS will resemble the sample below. ```css .heart-pulse { -webkit-animation: pulse 2s infinite; animation: pulse 2s infinite; } @-webkit-keyframes pulse { 0% { transform: scale(1); } 100% { transform: scale(1.5); } } @keyframes pulse { 0% { transform: scale(1); } 100% { transform: scale(1.5); } } ```
